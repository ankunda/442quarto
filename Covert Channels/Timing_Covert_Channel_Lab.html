In the previous lab you built a storage covert channel — a message hidden in file permission bits. In this lab you will build a timing covert channel: a TCP server that transmits a perfectly ordinary text message one character at a time, but encodes a secret message in the pauses between each character.

The overt message can be completely mundane — a weather report, a poem, a status message. The receiver, watching the arrival time of each character, accumulates bits and silently reconstructs the hidden message. An observer who does not know the encoding scheme sees only the text.

Because each character of the overt message carries only one bit of the covert message, low bandwidth is unavoidable: sending a single ASCII character (7–8 bits) requires transmitting at least 8 overt characters. This is a fundamental property of covert channels.

We will terminate the covert message with the sentinel string EOF (encoded in binary like any other string), so the receiver knows when the hidden message is complete.

1 Setting Up the Server

1.1 A Basic TCP Server

Start with a minimal Python TCP server using the built-in socket library. This version simply accepts a connection and sends two lines of text:

import socket

HOST = "127.0.0.1"  # localhost — no traffic leaves this machine
PORT = 1337         # pick an unprivileged port (> 1024) not in common use

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.bind((HOST, PORT))
s.listen(0)  # queue length 0: reject any connection while busy

print(f"Server is listening on {HOST}:{PORT}")

while True:
    c, addr = s.accept()
    print(f"Connection from {addr} established")

    c.sendall(b"Hello from the other side.")
    c.sendall(b"I must have called a thousand times")

    c.close()
💡 Test it now

Before adding the timing logic, test this basic server. In a separate terminal, connect with either of these tools:

  • telnet (plain text, unencrypted): telnet 127.0.0.1 1337
  • netcat (general-purpose network utility): nc 127.0.0.1 1337

Press Ctrl+C or type exit to disconnect from netcat. You should see the server's two lines appear in your terminal.

1.2 Encoding the Covert Message

Now we need to prepare the hidden message as a binary string. We will convert each character to its 8-bit binary representation, then concatenate everything into one long bitstream:

from binascii import hexlify

covert = "secret" + "EOF"  # append EOF sentinel so the receiver knows when to stop
covert_bin = ""

for char in covert:
    # hexlify:  ASCII → hex string  (e.g. 'A' → b'41')
    # int(..., 16): hex → integer   (e.g. '41' → 65)
    # bin(...):  integer → binary   (e.g. 65  → '0b1000001')
    # [2:]:      strip the '0b' prefix
    # zfill(8):  left-pad to 8 bits (e.g. '1000001' → '01000001')
    covert_bin += bin(int(hexlify(char), 16))[2:].zfill(8)

# Example: "ABC" → "010000010100001001000011"
print(covert_bin)

We also need two time constants — one for a short delay (bit 0) and one for a long delay (bit 1). These values are negotiated implicitly between sender and receiver; both must use the same constants:

ZERO = 0.025  # seconds — delay to signal a 0 bit
ONE  = 0.1    # seconds — delay to signal a 1 bit

The ratio between ZERO and ONE (here 4×) should be large enough that measurement jitter on the client side does not cause a misread. For a robust threshold, the client can treat any delay above (ZERO + ONE) / 2 = 0.0625 s as a 1.

1.3 The Full Server Loop

Putting it together: the server sends each character of the overt message using send() (not sendall()), then sleeps for the appropriate duration based on the next bit of the covert message. Once the overt message is exhausted, it sends the EOF sentinel and closes:

import socket
from time import sleep
from binascii import hexlify

HOST = "127.0.0.1"
PORT = 1337
ZERO = 0.025
ONE  = 0.1

# --- Prepare covert bitstream ---
covert = "secret" + "EOF"
covert_bin = ""
for char in covert:
    covert_bin += bin(int(hexlify(char), 16))[2:].zfill(8)

# --- Prepare overt message (must be at least as long as covert_bin) ---
overt_msg = ("The weather today is partly cloudy with a high of 72 degrees. "
             "Winds from the southwest at 10 mph. Have a pleasant afternoon.")

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.bind((HOST, PORT))
s.listen(0)
print(f"Server listening on {HOST}:{PORT}")

while True:
    c, addr = s.accept()
    print(f"Connected: {addr}")

    n = 0
    for char in overt_msg:
        c.send(char.encode())          # send one character of the overt message
        bit = covert_bin[n % len(covert_bin)]
        sleep(ZERO if bit == "0" else ONE)  # encode the covert bit in the delay
        n += 1

    c.send(b"EOF")   # signal end of covert message
    c.close()
    print("Connection closed.")

2 Setting Up the Client

⚠️ Windows / VM Timing Note

Timing covert channels depend on accurate sleep and clock measurements. These have historically been less reliable on Windows and on Linux VMs hosted on Windows. If your measurements are inconsistent, try widening the gap between ZERO and ONE, or run on a native Linux machine.

2.1 A Basic Receiver

Start with the simplest possible client — connect, receive everything, print it:

import socket

HOST = "127.0.0.1"
PORT = 1337

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

try:
    s.connect((HOST, PORT))
    print("Connected to server.")

    data = s.recv(1024)
    print("Received:", data.decode("utf-8"))
finally:
    s.close()

2.2 Receiving One Character at a Time

The basic client may buffer incoming data and display it in chunks rather than character-by-character. This breaks our timing measurements. We need to:

  • 1

    Use sys.stdout.write() instead of print() so each character is written immediately.

  • 2

    Call sys.stdout.flush() after each write to force the output buffer to empty.

  • 3

    Loop, receiving one chunk at a time, until the EOF sentinel is received.

import sys
import socket

# ... (connection setup as above) ...

data = s.recv(4096)
while data.rstrip(b"\n") != b"EOF":
    sys.stdout.write(data.decode("utf-8", errors="replace"))
    sys.stdout.flush()
    data = s.recv(4096)   # wait for the next character

2.3 Measuring Delays

To decode the covert bits, we measure the elapsed time between receiving successive characters. The time() function returns the current time as a float in seconds:

from time import time

ONE  = 0.1     # must match the server's ONE constant
THRESHOLD = 0.0625  # midpoint between ZERO (0.025) and ONE (0.1)

covert_bin = ""

t0 = time()               # timestamp before receiving anything
data = s.recv(4096)
t1 = time()               # timestamp after receiving something

delta = round(t1 - t0, 3) # elapsed time, rounded to milliseconds

# Classify: if delta >= threshold, it was a long delay (1); otherwise short (0)
if delta >= THRESHOLD:
    covert_bin += "1"
else:
    covert_bin += "0"

The threshold approach is more robust than comparing directly to ONE, because network and system jitter can nudge the measured delay away from the exact intended value.

2.4 Decoding the Bit String

Once you have accumulated enough bits (stop when you see the EOF sentinel decoded), convert them back to characters. Process 8 bits at a time:

from binascii import unhexlify

covert = ""
i = 0

while i < len(covert_bin):
    byte = covert_bin[i : i + 8]         # take 8 bits
    n = int("0b" + byte, 2)              # binary string → integer
    try:
        covert += unhexlify(f"{n:02x}") .decode("ascii")
    except Exception:
        covert += "?"   # bit error — character could not be decoded

    if covert.endswith("EOF"):           # stop when sentinel is found
        covert = covert[:-3]            # strip the sentinel
        break

    i += 8

print("\n\n[Covert message]:", covert)

3 Your Challenge

🎯 Lab Challenge

The server code above is complete. The client section has given you all the necessary components, but they are deliberately not assembled into a single working script. Your job is to combine them into a client that:

  • Connects to the server
  • Receives the overt message character by character
  • Measures the inter-character delay and classifies each as 0 or 1
  • Accumulates the bit string and decodes it into ASCII
  • Stops when it detects the EOF sentinel
  • Prints the recovered covert message

The flow through your client should be:

Step 1
Connect to server
→
Step 2
Recv char + measure delay
→
Step 3
Classify bit (0 or 1)
→
Step 4
Decode when 8 bits ready
→
Step 5
Stop at EOF sentinel

A few things to keep in mind as you build your solution:

  • →

    Threshold vs. exact comparison. Don't compare delta == ONE. Use the midpoint threshold (THRESHOLD = (ZERO + ONE) / 2) to handle timing jitter gracefully.

  • →

    Buffer flushing. Use sys.stdout.write() + sys.stdout.flush() to see the overt message appear in real time as the server sends it.

  • →

    Error handling. The ? substitution for undecodable characters is a useful debugging hint: too many question marks indicate that your threshold is wrong or timing is unreliable.

  • →

    Socket cleanup. Always close the socket in a finally block so the port is freed even if an exception occurs.

Good luck!
